# Telegram Local AI Agent (로컬 AI 에이전트 제어 봇)

Study 워크스페이스 내에서 텔레그램 메신저를 통해 강력한 AI 에이전트(Gemini 2.5 Flash)와 대화하며, 워크스페이스 내의 파일 읽기/쓰기, 디렉토리 탐색 및 쉘 명령어 실행을 원격에서 수행할 수 있는 스마트 봇 프로젝트입니다.

이 봇은 기존의 단순 키워드 매핑 방식에서 벗어나 **Gemini Function Calling(도구 사용) 기술**을 적용하여, AI 에이전트가 사용자님의 자연어 요청에 따라 로컬 PC 내의 파일과 명령을 유기적으로 제어하는 진정한 **AI 에이전트**로 작동합니다.

---

## 주요 기능

1. **파일 읽기 및 수정 (`read_file`, `write_file`)**:
   - 텔레그램 채팅방에서 자연어로 파일의 내용을 물어보거나 수정을 지시할 수 있습니다.
   - *예시*:
     - `"README.md 파일 읽어서 요약해줘"`
     - `"index.js 파일에서 도움말 문구를 좀 더 귀엽게 바꿔줘"`
     - `"telegram-ai-bot 폴더 아래에 memo.txt 파일을 새로 만들고 '내일 할 일 리스트'를 적어줘"`
2. **디렉토리 탐색 (`list_directory`)**:
   - 로컬 작업 영역의 구조를 물어보고 탐색을 요구할 수 있습니다.
   - *예시*: `"WSOP-Web 폴더 밑에 어떤 서브디렉토리가 있는지 확인해줘"`
3. **로컬 명령어 실행 (`execute_command`)**:
   - 백그라운드 서버 기동 또는 동기식 스크립트(테스트, 크롤링) 실행을 자유롭게 요청할 수 있습니다.
   - *예시*:
     - **대시보드 기동(백그라운드)**: `"대시보드 서버 켜줘"`, `"웹 실행해"` (AI가 백그라운드 모드로 명령을 즉시 기동하고 응답함)
     - **크롤러 실행**: `"크롤러 빠른 검증으로 실행해줘"`, `"선수 데이터 10명 풀스캔 돌려줘"`
     - **QA 자동화 테스트**: `"QA 테스트 돌리고 결과 리포트해줘"`
4. **AI 일반 대화 및 멀티턴 지원**:
   - 대화 세션별로 맥락(Context)이 보관되므로 앞선 대화나 작업 결과를 기억하여 연속적인 지시를 처리합니다.
   - *예시*: `"방금 수정한 코드로 테스트 다시 돌려봐"`
5. **수동 슬래시 명령어**:
   - 명시적인 명령어 전송이 필요할 때를 대비하여 슬래시 명령어도 유지됩니다.
   - `/run`, `/run_all`, `/run_crawler`, `/run_crawler_full`, `/run_qa`, `/help`

## 폴더 구조

```text
Study/
├── telegram-ai-bot/          <- 본 프로젝트 폴더
│   ├── .env                  <- API 토큰 정보 파일 (TELEGRAM_BOT_TOKEN, GEMINI_API_KEY)
│   ├── index.js              <- Function Calling 루프를 포함한 AI 에이전트 메인 코드
│   ├── package.json          <- 프로젝트 메타데이터 및 의존성
│   ├── Start-Bot.bat         <- 봇 구동용 원도우 배치 파일
│   └── README.md             <- 본 문서

    ├── WSOP-Player-Standings-Crawler/
    └── WSOP-Web-Automation/
```

## 실행 방법

봇이 Playwright 자동화 테스트(Headed 브라우저 기동 등) 및 대시보드 서버를 백그라운드 세션 제약 없이 데스크톱 GUI 세션에서 안전하게 띄울 수 있도록 하려면, **반드시 사용자님의 일반 윈도우 대화형 세션에서 실행**해야 합니다.

### 1. 봇 실행 방법 (배치 파일 이용)
1. `telegram-ai-bot` 폴더에 있는 **`Start-Bot.bat`** 파일을 마우스로 더블클릭합니다.
2. 콘솔창이 나타나며 의존성(`node_modules`)이 없을 경우 자동으로 다운로드한 뒤, 봇이 기동됩니다.
   - `🤖 텔레그램 AI 에이전트 제어 봇이 시작되었습니다. 메시지를 기다리는 중...` 메시지가 뜨면 성공입니다.
3. 봇을 켜두는 동안에는 이 콘솔창을 닫지 말고 **최소화**해 두세요. (창을 닫으면 봇이 종료됩니다.)

### 2. 보안 지침 및 제한 사항
- **보안 위험**: 이 봇은 원격에서 파일 편집 및 쉘 명령어 실행 권한을 AI에게 온전히 부여합니다. **텔레그램 봇 API 토큰 및 Gemini API Key가 외부에 노출되지 않도록 극히 보안에 주의하십시오.**
- **명령어 필터링**: 시스템에 직접적인 해를 가할 수 있는 명령(`del /s`, `rmdir /s`, `shutdown` 등)은 내부 코드 보안 필터에 의해 차단되며 실행되지 않습니다.
- **경로 제한**: 파일 편집 및 명령어 실행은 기본적으로 `Study` 워크스페이스 폴더 내부로만 상한선이 제한되어, 외부 시스템 핵심 파일 조작을 방지합니다.
