게임 개발 PM 업무 방향성

개발 PM의 역할을 단순 일정 관리가 아닌 조율, 의사결정, 품질 유지, 릴리즈 판단의 관점에서 정리한 인터랙티브 포트폴리오입니다. 역할 허브, 의사결정 구조, 스프린트 리듬, 핵심 역량 프레임워크를 한 화면 안에서 확인할 수 있도록 구성했습니다.

01 — Positioning

개발 PM의 역할 허브

개발 PM은 조직 내 여러 기능 조직의 중심에서 흐름을 맞추는 역할입니다. 각 노드를 선택하면 협업 포인트와 실제 관점을 확인할 수 있습니다.

개발 PM은 조직의 각 기능 조직을 하나의 프로젝트 흐름으로 연결하는 중심 조율자입니다. 일정 관리 자체보다, 어디에서 흐름이 끊기고 품질 리스크가 생기는지를 먼저 파악하고 조정하는 역할에 가깝습니다.

02 — Decision Making

개발 PM의 의사결정 구조

개발 PM의 판단은 단순 일정 조정이 아니라, 스펙·트레이드오프·릴리즈·방향 전환까지 포함합니다. 카드를 선택하면 실제 상황에서 어떤 판단을 내려야 하는지 볼 수 있습니다.

스펙 결정 (Spec)

무엇을 빌드할지를 정의합니다. 기획팀의 아이디어, 데이터팀의 유저 인사이트, 개발팀의 기술 제약을 종합해 PRD를 작성합니다. 기능 하나를 추가하는 것보다, 그것이 전체 경험과 로드맵에 어떤 의미를 갖는지 정리하는 과정입니다.

03 — Sprint Rhythm

2주 스프린트 사이클

게임 개발 PM의 전형적인 2주 리듬을 시각화했습니다. 특정 업무만 반복하는 것이 아니라, 기획·개발 협업·운영 대응·분석·커뮤니케이션이 동시에 흐릅니다.

04 — Capability Framework

게임 개발 PM 핵심 역량 프레임워크

개발 PM에게 필요한 역량을 5개 축으로 정리했습니다. 시장 이해, 기술 구조 파악, 데이터 기반 판단, 커뮤니케이션, 실행 관리가 균형 있게 연결되어야 합니다.
